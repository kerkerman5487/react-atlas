Avatar

Avatar component creates a circular area where an image, letter or icon/glyphicon can be presented. Great for user profiles and lists.

NOTE: children will always take precedence over props passed into component.


  <section>
    <h5>Avatars</h5>
    {/* background color change */}
    <Avatar style={{backgroundColor: 'deepskyblue'}} >
      <i className="fa fa-github"></i>
    </Avatar>
    {/* title prop gets truncated to 1st letter */}
    <Avatar title="Nathan" />
    {/* icon beats title */}
    <Avatar title="Nathan" icon={<i className="fa fa-github"></i>} />
    {/* image beats icon */}
    <Avatar 
      icon={<i className="fa fa-github"></i>}
      image="cat.jpg"
    />
    {/* image beats title */}
    <Avatar title="Javier" image="cat.jpg" />
    {/* child beats parameters */}
    <Avatar title="Nathan" image="cat.jpg">
      <i className="fa fa-github"></i>
    </Avatar>
    <Avatar title="Nathan" icon={<i className="fa fa-github"></i>}>
      <img src="cat.jpg"/>
    </Avatar>
    {/* child string gets truncated to 1st letter */}
    <Avatar>Nathan</Avatar>
    {/* child should be <i>, <svg>, <img>, or string */}
    <Avatar><i className="fa fa-github"></i></Avatar>
  </section>

Properties

PropertyTypeDescriptionDefault ValueExample
childrennodeChildren should be either a string, an icon/glyphicon, or an image tag. "SomeName", <SomeIcon />, <img src="/path/to/image.jpg"/>
classNamestringA css class name that will be appended to the wrapping div around the avatar.''
iconstring, elementFor displaying an icon/glphyicon. Normally these will be another component or an element with a class on it. '' <GithubIcon />, <i class="fa fa-github"></i>
imagestringPath to an image "http://path.to/an/image.jpg"
titlestringA string. Avatar will use First letter of the string. "Nathan" will output "N"
